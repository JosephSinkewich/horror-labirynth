# **Тестовое ТЗ для Unity-разработчика**

## **Цель задачи**

Нужно сделать небольшой прототип игры от первого лица в Unity.

Игрок появляется в небольшом видимом лабиринте, бегает по нему, собирает бриллианты и избегает врагов. Главная цель \- собрать все бриллианты и добраться до выхода.

Ориентировочная занятость: **3-6 часов**.

## **Общая идея**

Игра должна быть простой, но законченной по игровому циклу:

1. Игрок появляется в лабиринте.  
2. В лабиринте в рандомных точках спаунятся бриллианты.  
3. Игрок собирает бриллианты.  
4. По лабиринту двигаются враги.  
5. Если враг догоняет игрока \- игрок проигрывает.  
6. Если игрок собрал все бриллианты и дошёл до выхода \- игрок побеждает.

## **Камера и управление**

Игра от первого лица.

Игрок должен уметь:

* ходить вперёд/назад/влево/вправо;  
* вращать камеру мышью;  
* бегать при зажатии Shift;  
* не проходить сквозь стены.

Можно использовать стандартный CharacterController или собственную простую реализацию движения.

## **Лабиринт**

Лабиринт не обязательно делать процедурным.

Можно сделать вручную небольшую сцену:

* пол;  
* стены;  
* несколько проходов;  
* стартовая точка игрока;  
* зона выхода;  
* маршруты или зоны движения врагов.

Важно: лабиринт должен быть читаемым. Игрок не должен долго блуждать и не понимать, куда идти. Это не хардкорный maze, а короткая бродилка на 2–5 минут.

Допускается простой визуал из кубов/примитивов.

## **Бриллианты**

В лабиринте должно быть несколько бриллиантов, например 5–10 штук.

Требования:

* бриллиант должен быть видимым;  
* при касании игроком бриллиант собирается;  
* собранный бриллиант исчезает со сцены;  
* счётчик собранных бриллиантов обновляется в UI;  
* после сбора всех бриллиантов активируется возможность победы через выход.

Желательно, чтобы бриллиант немного вращался или визуально выделялся.

## **Враги**

В лабиринте должно быть минимум 2 врага.

Враг должен:

* перемещаться по лабиринту;  
* преследовать игрока, если игрок попал в радиус обнаружения;  
* догонять игрока;  
* вызывать поражение при контакте с игроком.

Можно сделать простую реализацию:

* враг патрулирует между точками;  
* если игрок близко \- враг начинает преследование;  
* если игрок далеко \- враг возвращается к патрулю.

## **Победа и поражение**

### **Победа**

Игрок побеждает, если:

* собрал все бриллианты;  
* дошёл до зоны выхода.

После победы нужно показать UI-экран:

* “Победа”;  
* количество собранных бриллиантов;  
* кнопка Restart.

### **Поражение**

Игрок проигрывает, если:

* враг коснулся игрока.

После поражения нужно показать UI-экран:

* “Поражение”;  
* кнопка Restart.

## **UI**

На экране должен быть минимальный UI:

* количество собранных бриллиантов, например  
* экран победы;  
* экран поражения;  
* кнопка перезапуска уровня.

UI достаточно сделать на примитивах.

## **Архитектурные требования**

При работе важно показать грамотный архитектурный подход к разработке. Желательно разделить ответственность между классами:

* отдельный класс для движения игрока;  
* отдельный класс для управления камерой;  
* отдельный класс для бриллианта;  
* отдельный класс для счётчика/коллекции бриллиантов;  
* отдельный класс для врага;  
* отдельный класс для состояния игры: победа/поражение/рестарт;  
* отдельный класс для UI.  
* какая-то точка входа и состояния игры  
* возможно фабрика по необходимости

Нельзя складывать всю логику в один большой GameManager.

Допустимо использовать простой GameManager/GameSession, но только как координатор, а не как класс, который делает всё.

## **Технические требования**

* Unity 6/C\#.  
* Проект должен запускаться без ошибок.  
* Сцена должна быть готова к запуску через Play.  
* Можно использовать Unity primitives.  
* Можно использовать бесплатные ассеты,но строго по желанию.   
* Код должен быть читаемым.

## **Что нужно предоставить**

Разработчик должен предоставить:

1. Unity-проект или архив проекта.  
2. Готовую сцену с лабиринтом.  
3. Исходный код.  
4. Краткое описание управления.  
5. Краткое описание архитектуры:  
   * какие классы за что отвечают;  
   * где находится логика победы/поражения;  
   * где находится логика сбора бриллиантов;  
   * где находится логика врагов.

## **Критерии приёмки**

Работа считается выполненной, если:

* игрок может ходить по лабиринту от первого лица;  
* камера нормально вращается;  
* стены блокируют движение игрока;  
* бриллианты собираются;  
* UI показывает прогресс сбора;  
* враги двигаются;  
* враги могут поймать игрока;  
* при контакте с врагом наступает поражение;  
* после сбора всех бриллиантов можно победить через выход;  
* есть экран победы;  
* есть экран поражения;  
* есть рестарт;  
* код не свален в один огромный скрипт;  
* проект запускается без ошибок.

## **Что будет оцениваться**

В первую очередь оценивается не визуал, а:

* понятная структура кода;  
* разделение ответственности;  
* аккуратная работа с Unity-компонентами;  
* умение сделать законченный игровой цикл;  
* отсутствие хаоса в зависимостях;  
* читаемость классов и методов;  
* способность за ограниченное время сделать рабочий прототип.

Визуал может быть простым. Главное \- чтобы игра была играбельной и код можно было поддерживать
